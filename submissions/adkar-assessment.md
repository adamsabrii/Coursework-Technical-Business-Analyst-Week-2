## Practical risks and mitigations

What could go wrong if this area isn't addressed?

Flags:

\+ backlog item

% reporting requirement

All can be executive briefing messages

### Agent (reinforcement)

If the new system has cracks and doesn't function as intended or simply shuffles problems around then agent adoption will fall and agent workload won't be reduced.

Mitigations: run a pilot with real agents and sample of customers. + % 

### Customer (ability)

The platform needs to be user-friendly and easy to use for the average customer to adopt its features.

Mitigations:
* User testing with real customers on the Figma prototype, iterate design based on feedback. + % 
* Tutorial/ guidance content on using the new platform. +
* Support system. +
* Analytics and monitoring after launch to see customer activity. + %

### Customer (reinforcement)

If the platform isn't intuitive then customer will resort to overloading agent inboxes and phone lines

Mitigations:
* Deliberately make it more difficult to phone agents. + %
* A well documented support page. +
* Support AI chatbot. +
* Page feedback showing user progress. + %

### Finance and compliance (ability)

The system must capture journey tracking, and key financial changes.

Mitigations:
* Data audit before launch. + %
* Monitoring dashboards with changeable views to group by key metrics. + %

### Finance and compliance (reinforcement)

If the automated system or agents don't effectively capture case observability then finance will have to spend effort to fill in gaps.

Mitigation:
* Incentives for accountability. + %
* Alerts that flag when logging fails or is incomplete. + %
* Regular audits to catch data gaps early. + %
